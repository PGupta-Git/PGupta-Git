# GitHub Landing Page Checklist

This checklist is for turning the profile into a recruiter-friendly landing page.

## 1) Pin the right repositories

GitHub only allows 6 pinned repos. Recommended pinned set for a DS (analysis/experimentation) profile:

1. `case-study-player-availability-decision-support`
2. `case-study-tactical-recruitment-performance-analysis`
3. `case-study-drill-design-app`
4. `open-football-experimentation-lab`
5. `Gupta_et_al_RST_Paper_Submission`

How:

- Go to your profile -> "Customize your pins" -> select the repos above.

## 2) Add evidence artifacts (this is the difference-maker)

Each docs-only case study repo has an `assets/` folder and README placeholders.

Add 3-5 redacted images per repo:

`case-study-player-availability-decision-support`: calibration plot, longitudinal trend, feature contribution summary, decision card screenshot.

`case-study-tactical-recruitment-performance-analysis`: player radar, shot/chance map, benchmark distributions, scouting report page.

`case-study-drill-design-app`: editor screenshot, drill library, sharing view, billing/subscription view.

Tip: if you can't use real screenshots, create "representative" images with blurred labels and clearly label them as mock examples.

## 3) Social preview images

For each repo, set a social preview image (GitHub repo Settings -> Social preview).
Choose one graphic that makes sense when seen small in a grid.

## 4) Profile fields (quick credibility wins)

- Profile headline: DS (analysis/experimentation)
- Location (optional)
- Link: LinkedIn + a resume link if you have one
- Optional: enable "Private contributions" to show activity without exposing code
